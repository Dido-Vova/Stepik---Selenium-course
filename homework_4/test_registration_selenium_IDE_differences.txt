Простой тест, возможно не очень сильно видны отличия использования IDE и самописного кода.

Что сделал Selenium IDE из того, что не сделал я:
1. Импортировал множество неиспользуемых библиотек. Думаю, это легко настраивается. Не буду относить к недостаткам.
2. Создал Setup и Teardown методы в той же сущности. Я создал одну сущность для всей папки.
3. Задал размер окна открывшейся страницы. Скорее плюс IDE.
4. Установил курсор в поле ввода Email. Тоже плюс IDE.

Что сделал я, из того, что не сделал IDE
1. Вынес в фикстуры открытие и закрытие браузера с заданными параметрами, в соответствии с планом тестирования.
2. Вынес локаторы в отдельный класс.
3. Сделал проверку ожидаемого результата assert-ом.

Результат.
Плюсы использования IDE:
- Правильно подобрал локаторы (по крайней мере на этом простом примере).
- Записал клики мышью, которыми я пренебрег.

Минусы:
- Для параметризации и циклического запуска - подходит только часть кода. Остальное нужно менять.
- Записывает движения человека без какой-либо интеллектуальной обработки/оптимизации кода.
- Не использует переменные. Все выполняет в одну строку. Возможно из-за того, что очень простые действия.

Заключение - IDE может использоваться для ускорения написания некоторых рутинных тестов. 
Только часть кода пригодна для использования вместе с набором подобных тестов для запуская с разными параметрами и сценариями.
